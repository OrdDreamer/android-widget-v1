# Android Indie App Launch Framework

## Про цей набір промптів

Цей набір призначений для solo Android-розробника, який хоче:
- підготувати застосунок до Google Play;
- пройти модерацію без критичних помилок;
- оптимізувати сторінку магазину;
- отримати перших органічних користувачів;
- побудувати основу для розвитку продукту.

---

# Як використовувати

Не запускай всі промпти одночасно.

Правильний порядок:

1. Product Discovery
2. Google Play Release Preparation
3. ASO & Branding
4. Growth Marketing
5. Final Audit


# Промпт 1
# Product Discovery та позиціонування

Ти - Senior Mobile Product Manager з досвідом запуску Android indie-додатків у Google Play.

Твоя задача - допомогти мені правильно сформувати продукт перед релізом: зрозуміти цільову аудиторію, проблему користувача, позиціонування та конкурентні переваги.

Я розробляю Android-додаток:

[ОПИС ДОДАТКА:
Photo Widget — мінімальний Android-додаток, чия єдина мета: показати одне обране фото з галереї на робочому столі у вигляді змінюваного за розміром віджета, з контролем вигляду й поведінки кліку. Кілька віджетів працюють незалежно один від одного.

Суть
Користувач додає віджет на home screen, обирає фото через системний Photo Picker, налаштовує як воно виглядає (масштаб, форма, поворот, позиція) і що відбувається при натисканні. Додаток сам по собі — менеджер цих віджетів: список, редагування, скидання, pin на стіл.

Офіційний опис: «Показує обране фото з галереї на робочому столі».

Для кого / вимоги
Android 13+ (API 33), target API 35
Без повного доступу до сховища — лише Photo Picker
Що вміє користувач
У додатку (головний екран)
Додати на робочий стіл — якщо лаунчер підтримує requestPinAppWidget
Список активних віджетів з:
кастомною назвою або Віджет #N
розміром у клітинках (2x2 тощо)
мініатюрою фото
Редагувати будь-який віджет
Скинути — очищає фото й налаштування, віджет лишається на столі
Клік по віджету може одразу відкрити редагування саме цього екземпляра
При додаванні віджета
Системне вікно конфігурації (APPWIDGET_CONFIGURE) — той самий екран налаштувань, що й у додатку.

Налаштування вигляду
Опція	Варіанти
Фото
Photo Picker (лише зображення)
Назва
підпис у списку додатку (до 40 символів)
Поворот
±90° (разом з EXIF-орієнтацією)
Позиціонування
сітка 3×3 (центр / сторони / кути)
Масштаб
Cover (обрізає) / Contain (вміщає з полями)
Форма
прямокутник / закруглений / коло
Радіус кутів
0…48 dp (лише для закругленого)
Клік
без дії / відкрити додаток / відкрити цей віджет
Без фото клік завжди відкриває налаштування цього віджета — навіть якщо збережено іншу дію.

Розмір на робочому столі
Віджет можна розтягувати (горизонтально й вертикально). При зміні розміру або орієнтації зображення перерендерюється під новий розмір. Мін. ~110×110 dp, типовий target — 2×2 клітинки.

Життєвий цикл віджета
Додавання → конфіг → RESULT_OK → оновлення на столі
Зміна опцій / resize → офлайн-рендер фото в bitmap → показ у лаунчері
Видалення з лаунчера → конфіг цього id стирається
Періодичного auto-update немає (updatePeriodMillis = 0)
Технічно фото доставляється через setImageViewBitmap (не через URI/FileProvider) — це критично для стабільності на Android 17+.

Що зберігається
На кожен appWidgetId у DataStore: URI фото, поворот, alignment, scale, shape, радіус, click action, display name, стабільний номер #N. Номери не «з’їжджають», якщо видалити інші віджети.

Чого немає
Слайдшоу, кілька фото, відео, камера, URL-зображення
Lock-screen / Glance-віджети
Видалення віджета з лаунчера зсередини додатку (лише скидання конфігу)
UI для «налаштувань за замовчуванням» (у репозиторії API є, екрана немає)
Технології (коротко)
Kotlin, Jetpack Compose + Material 3, AppWidgetProvider + RemoteViews, DataStore Preferences, Coil (прев’ю), ExifInterface (орієнтація з камери).

Підсумок: це не галерея і не соцмережа — це тонкий шар керування фото-віджетами на home screen: одне фото на віджет, гнучкий вигляд, незалежні екземпляри, простий менеджер у застосунку.
]
